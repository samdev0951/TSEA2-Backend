### Functions

Functions are defined using the keyword ‘def’. After the keyword, you write the function name and include parentheses, and colons after. The following lines of code should be indented, as it’s an indicator of where the line of code belongs. The indented lines of code under a function declaration only run if the function name is explicitly called - for example, by writing function_name() and supplying any required parameters.

Function naming - There are different ways of writing the name of a function. The first way to write a function name is snake casing, where there’s an underscore between each word, e.g., my_function(). The second one is camel casing, where the first word of the function name is written in lower case, but the first letter of the following words is capitalised, e.g., myFunctionIsVeryNice().

### Modules

Modules are separate files, specifically made for storing functions that might be used across several different files. This helps reduce repetition by having only one declaration of a function and using methods like ‘import’ to use a module-defined function from the module file.
